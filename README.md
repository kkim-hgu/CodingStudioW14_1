# 15-1. 다윗관 관리 프로그램 - 호실변경기능 업데이트하기

**기본설명**

다윗관이라고 불리는 가상의 생활관에는 외국인 학생들이 거주한다. 이 곳은 2층으로 되어 있으며, 1층에는 여학생, 2층에는 남학생이 거주하게 된다. 각 층마다 1호실부터 15호실까지 15개의 2인1실의 방이 있다.

이 프로그램은 다윗관에 거주하는 학생들의 정보를 관리한다. 관리하는 정보는 다음과 같다.
학생이름, 성별(F or M), 입학년도(2015 ~ 2018), 방호실(101 ~ 215), 벌점(0 ~ 10)

기본적으로 프로그램이 시작되면 members.txt로부터 40명의 관생정보를 읽어온 후 (완성 코드 제공), 5개의 메뉴를 선택하면서 필요한 기능이 수행된다.

[1] 목록 - 현재의 모든 관생 정보를 화면에 출력한다. --> 이 기능은 코드에 이미 구현되어 제공한다.
[2] 호실변경 - 특정 학생의 호실을 변경한다. --> 이 기능은 코드에 일부만 구현되어 제공한다.
[3] 벌점부여 - 특정 학생에게 벌점을 1점 부과한다. --> 이 기능은 직접 구현해야 한다.
[4] 검색 - 이름의 일부(2글자이상)만으로 관생정보를 검색한다. --> 이 기능은 코드에 일부만 구현되어 제공한다.
[5] 보고서 저장 - 현재의 다윗관 현황을 파일에 저장한다. --> 이 기능은 직접 구현해야 한다.


**문제설명 : 호실변경기능 업데이트하기**


제공된 코드에서의 호실변경기능은 학생이름을 입력받아 해당 학생의 정보가 있는 경우, 무조건 새로운 호실을 입력받아 수정하고 있다. 아래 실행 예처럼 호실이 없는 333호를 배정하는 엉뚱한 작업도 허용하고 있다.


```
40명이 저장된 파일을 읽었습니다.

[1]목록 [2]호실변경 [3]벌점부여 [4]검색 [5]보고서 저장 [0]종료 >> 2
호실 변경하려는 학생 이름은? >> David
이름 성별 학번 호실 벌점
David [M] 2016 209 0
새로운 호실을 입력하시오. >> 333
수정완료!

[1]목록 [2]호실변경 [3]벌점부여 [4]검색 [5]보고서 저장 [0]종료 >> 0
```


따라서, 다음의 사항을 고려하여 올바른 호실이 입력될 때까지 계속해서 호실을 입력도록 기능을 업데이트하라.

1. 새로운 호실을 입력했을 때에 이미 이전의 호실을 입력하면 그대로 저장하고 수정완료로 처한다.
2. 다윗관은 101호~115호, 201호~215호가 있으므로, 이에 해당되지 않은 호실은 배정불가(호실 오류)로 처리한다.
3. 여학생은 1층, 남학생은 2층만 배정해야 하므로, 해당 학생의 성별을 고려하여 다른 층 호실을 입력하면 배정불가(층 오류)로 처리한다.
4. 각 호실에는 최대 2명까지만 입실할 수 있으므로, 이미 2명이 채워진 호실을 입력하면 배정불가(인원초과)로 처리한다.



실행 예)
```
40명이 저장된 파일을 읽었습니다.

[1]목록 [2]호실변경 [3]벌점부여 [4]검색 [5]보고서 저장 [0]종료 >> 2
호실 변경하려는 학생 이름은? >> David
이름 성별 학번 호실 벌점
David [M] 2016 209 0
새로운 호실을 입력하시오. >> 101
배정불가(층 오류)
새로운 호실을 입력하시오. >> 333
배정불가(호실 오류)
새로운 호실을 입력하시오. >> 211
배정불가(인원초과)
새로운 호실을 입력하시오. >> 201
수정완료!

[1]목록 [2]호실변경 [3]벌점부여 [4]검색 [5]보고서 저장 [0]종료 >> 0
```

# 15-2. 다윗관 관리 프로그램 - 벌점부과기능 구현하기


**문제설명 : 벌점부과기능 구현하기**


다음의 순서대로 특정 학생에게 벌점 1점을 부과하는 기능을 구현하라.

1. 벌점을 부여할 학생 이름을 묻는다.
2. 목록에서 해당 학생의 이름이 없으면 "없는 학생입니다."를 출력한다.
3. 있는 경우 해당 학생의 정보를 출력하고, 벌점 부과여부를 묻는다. 이때, YES는 1을, NO는 0을 입력받는다.
4. 1을 입력받으면, 해당 학생에게 벌점을 1점 부과한 후, 변경된 벌점을 출력한다.

실행 예)

```
40명이 저장된 파일을 읽었습니다.
```
